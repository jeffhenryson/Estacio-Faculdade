Como usar
- Clonar: git clone <URL-do-repositorio>
- Organizar conteúdos: crie pastas seguindo a estrutura sugerida e mantenha nomes descritivos.
- Entregas: mantenha um README por projeto/lab explicando objetivo, requisitos e instruções de entrega.

Contribuição
- Siga convenções simples: nomes de pastas claros, README por item quando necessário.
- Envsios: crie pull requests com descrições curtas das alterações.
- Use commits atômicos e mensagens claras (ex.: feat: adicionar laboratório 01; fix: corrigir README).
